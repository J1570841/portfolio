Spaceship Titanic

Descripción del proyecto: En una asignatura universitaria, participé en un proyecto individual en formato datathon, en 3 días se tenia que resolver un problema de clasificación, de una competición kaggle, con los datos de este link.

Mi rol: Trabajé individualmente para resolver el problema y quedar lo más arriba posible, utilizando Jupyter Notebook y Python. Mi responsabilidad principal fue analizar los datos y hacer un modelo predictivo con la mayor accuracy possible.

Tecnologías usadas:

  • Lenguajes: Python
  • Plataformas: Jupyter Notebook, kaggle para descargar los datasets y subir la solución.
  • Colaboración: Git para control de versiones.
  • Bibliotecas usadas:
    • Pandas: Usada para el tratamiento de los distintos excels.
    • NumPy: Usada para tratamiento de los datos de distintos excels.
    • Matplotlib: Usada para dibujar gráficos a partir de los datos tratados de los excels.
    • Sklearn: Usada para tratar los datos, como manejar datos faltantes, estandarizar caracteristicas, reducir dimensionalidad, dividir el conjunto de datos en conjuntos de entrenamiento y prueba de manera eficiente...
    • Warnings: Usada para evitar que salieran warnings del tipo "Future Warning"

Enlace al repositorio: El código está almacenado en un repositorio privado. Si se desea acceder al código, puedo compartirlo durante una entrevista.

Equipo: Trabajo individual.

Resultados: Con el modelo entrenado según modelo SVM, se logró un score del 0,77507.

F1

Descripción del proyecto: En una asignatura universitaria, participé en un proyecto individual en el cual intente hacer dos modelos para predecir cuantos coches iban a acabar una carrera y una carrera de sprint en la f1 a partir de los datos de este link.

Mi rol: Trabajé individualmente para resolver el problema, utilizando Jupyter Notebook y Python. Mi responsabilidad principal fue juntar todos los archivos que habia en 2 datasets, analizar los datos y hacer dos modelos predictivos (uno para carreras al sprint y otro para carreras normales) con la mayor accuracy possible.

Tecnologías usadas:

  • Lenguajes: Python
  • Plataformas: Jupyter Notebook, kaggle para descargar los datasets y subir la solución.
  • Colaboración: Git para control de versiones.
  • Bibliotecas usadas:
    • Pandas: Usada para el tratamiento de los distintos excels.
    • NumPy: Usada para tratamiento de los datos de distintos excels.
    • Matplotlib: Usada para dibujar gráficos a partir de los datos tratados de los excels.
    • Seaborn: Usada para dibujar gráficos estadísticos de alto nivel para realizar análisis exploratorio de datos.
    • Sklearn: Usada para tratar los datos, como manejar datos faltantes, estandarizar caracteristicas, reducir dimensionalidad, dividir el conjunto de datos en conjuntos de entrenamiento y prueba de manera eficiente...
    • Warnings: Usada para evitar que salieran warnings del tipo "Future Warning"
    • Folium: Usada para crear un mapa de la ubicación de los circuitos de los datasets.

Enlace al repositorio: El código está almacenado en un repositorio privado. Si se desea acceder al código, puedo compartirlo durante una entrevista.

Equipo: Trabajo individual.

Resultados: Con los modelos entrenados según Gradient Boosting, se hizo una validación cruzada para ambos modelos en las cuales se pudo ver que habia overfitting, aunque se intento mitigar mediante Recursive Feature Elimination (RFE), se logró reducir un poco el overfitting, aunque aún estaba presente.
