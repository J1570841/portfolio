Proyectos Android

FoodFeed

Descripción del proyecto: En una asignatura universitaria, participé en un proyecto en equipo para desarrollar una aplicación Android que, a través de inteligencia artificial, permite a los usuarios identificar alimentos mediante imágenes y obtener información nutricional, como calorías, proteínas, grasas y carbohidratos, utilizando APIs de terceros.

Mi rol: Trabajé junto a tres compañeros en la implementación del frontend de la aplicación, utilizando Android Studio y Java. Nos encargamos de:

  • Diseñar e implementar la interfaz de usuario (UI), garantizando que fuera intuitiva y fácil de usar.
  • Desarrollar las distintas pantallas de la aplicación.
  • Integrar la app con componentes de Android, como la cámara para la toma de fotos de los alimentos.
  • Colaborar en la integración de la aplicación con un backend, utilizando un webserver para almacenar datos en una base de datos y realizar consultas a las APIs que utilizamos.

Tecnologías utilizadas:

  • Lenguaje: Java
  • Plataforma: Android Studio
  • Colaboración: Git para control de versiones y trabajo en equipo.
  • Gestión de Tareas: Jira para la planificación y seguimiento de tareas del proyecto.
  • Metodología: Scrum
  • API: Food Data Central API

Enlace al repositorio: El código está almacenado en un repositorio privado. Si se desea acceder al código, puedo compartirlo durante una entrevista.

Equipo: Trabajo en equipo con otros 7 compañeros, divididos en 2 backend, 2 IA y 4 frontend.

BookPixel

Descripción del proyecto: En una asignatura universitaria, participé en un proyecto en equipo para desarrollar una aplicación Android que, mediante imagen, texto o voz, permite a los usuarios buscar información sobre libros y videojuegos, además de incluir un chatbot. El proyecto hizo uso de APIs de terceros.

Mi rol: Trabajé junto a dos compañeros en varias tareas del desarrollo, utilizando Android Studio, Java, Node.js, Typescript y Firebase. Mis principales contribuciones fueron:

  • Diseñar e implementar las Cloud Functions en Firebase.
  • Desarrollar varias pantallas de la aplicación, incluyendo la pantalla principal, las de búsqueda (imagen, texto, voz), y las pantallas de resultados, información del producto, carrito y post compra, historial, la lista de deseos y ajustes para cambiar el idioma.
  • Añadir soporte de internacionalización (i18n) (español, inglés, alemán, portugués, italiano, catalán, francés, gallego y vasco)
  • Integrar componentes de Android, como la cámara y microfóno para las búsquedas por imagen y voz.
  • Colaborar con la integración entre la aplicación y Firestore y la conexión entre app y las Cloud Functions
  • Investigar, seleccionar e integrar las APIs necesarias.

Tecnologías usadas:

  • Lenguajes: Java para la aplicación y Typescript para las cloud functions
  • Plataformas: Android Studio, Firebase, Google Cloud y Visual Studio Code
  • Colaboración: Git para control de versiones y trabajo en equipo.
  • APIs: Google Books API, RAWG API, Google Traductor API, Google Cloud Vision, Google Cloud Speech API.

App DR72

Descripción del proyecto: En Gometrics, participé en el desarrollo de una aplicación Android que, a través de conectividad Bluetooth, permite a los usuarios visualizar en formato gráfico y textual los valores que lee el dispositivo DR72, creado por la empresa.

Mi rol: Trabajé principalmente de manera individual, con apoyo puntual de los desarrolladores del hardware del DR72 para la implementación de la conectividad Bluetooth. Utilicé Android Studio y Java. Mi responsabilidad principal fue desarrollar la aplicación completa, desde la interfaz de usuario hasta la comunicación con el dispositivo.

Tecnologías usadas:

  • Lenguaje: Java
  • Plataforma: Android Studio
  • Colaboración: Git para control de versiones.

Enlace al repositorio: El código está almacenado en un repositorio privado y no puede ser compartido.

Equipo: Trabajo individual, con pequeñas colaboraciones por parte de los desarrolladores del Hardware para la parte de conectividad.
